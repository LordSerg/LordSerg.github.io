Reverse Polish Notation

Introduction:

Reverse Polish Notation (RPN) is a mathematical notation scheme to represent math equasion. It is differs from the traditional infix notation used in most mathematical expressions. Example of traditional infix notation:

a * (b + c)

In RPN, operators ('+' '-' '*') are placed after their operands ('a' 'b' 'c'), eliminating the need for parentheses or operator precedence rules, like this:

a b c + *

This unique notation offers simplicity, efficiency, and clarity in mathematical calculations. It also allows for a straightforward evaluation process and eliminates the need for brackets or operator precedence rules.

So how to built that representation?

You will need:

  • an input string array, where each element is operator or operand
  • an answer-stack
  • a temporary-stack

Algorythm itself:

- While there are symbols to read:

+ Read next symbol

+ if symbol == number or an postfix function (such as factorial) -> add it to the answer-stack

+ if symbol == prefix function (such as sin) -> add it to the temporary-stack

+ if symbol == opening bracket -> add it to the temporary-stack

+ if symbol == closing bracket ->

1) move top-element from temporary-stack to the answer-stack, until top element == open bracket. Herewith open bracket is removed from temporary-stack and NOT added to the answer-stack

+ if symbol == binary operation X (such as +) ->

1) move top-element from temporary-stack to the answer-stack while it is:

a) prefix function

b) its' priority is higher or equal to the priority of X

2) add X to the temporary-stack

- When there is no symbols to read - push every element from temporary-stack to the answer-stack
